# Product Demo Video Platform

Cinematic enterprise product demo video system — **Remotion + React + TypeScript + Playwright**.  
First project: **Rheem** (Water & HVAC Solutions). Add future apps in `projects/`.

---

## Repository layout

```
core/                    Shared platform (all projects reuse this)
  types.ts               ProjectConfig, ThemeConfig, content types
  themes/                Dark theme + resolveTheme() engine + ThemeContext
  branding/              BrandingEngine (injects theme), LogoRenderer
  components/            AnimatedBackground, ScreenShowcase, KPICounter, …
  scenes/                Config-driven: SharedIntro, SharedOutro, ProblemScene, …
  utils/                 animations, easing, transitions
  hooks/                 useAnimatedValue, useStagger, useTypewriter

automation/              Playwright recording system (project-aware)
  record.ts              Auto-explores app, records workflow clips
  sync.ts                Scans public/ → updates clipManifest + videoConfig
  validate.ts            Checks all clips exist before render

projects/
  rheem/                 Rheem project
    config/
      project.config.ts  ALL Rheem content: product info, scenes, features, metrics…
      segmentDefs.ts     Recording segments (login, dashboard, analytics, dispatch)
    composition.tsx      Rheem Remotion composition — wires scenes + config
    clipManifest.json    Generated by sync — maps clips→segments with real durations

render/
  validate.ts            Pre-flight asset check
  pipeline.ts            Full record→sync→validate→render in one command

src/
  Root.tsx               Remotion registry (registers project compositions)
  index.tsx              Remotion entry point (unchanged)

public/
  assets/                Legacy/manual recordings (Login.mp4 lives here)
  projects/rheem/recordings/   Playwright-generated clips
```

## End-to-end workflow

### 1. Configure credentials

```bash
cp .env.example .env
# Set:  RHEEM_APP_URL, APP_USERNAME, APP_PASSWORD
```

### 2. Record

```bash
# Provide URL + credentials — Playwright does the rest
npm run record -- https://your-app.com --user admin@co.com --pass secret

# Or use .env:
npm run record:rheem
```

Playwright opens Chromium, logs in, crawls nav links, and saves one MP4 per
workflow/page to `public/projects/rheem/recordings/`.  
**Manual assets always win** — drop any `.mp4` into `public/assets/` or
`public/projects/rheem/recordings/` and they'll be used instead.

### 3. Sync

```bash
npm run sync            # scans assets → updates clipManifest.json + videoConfig.json
npm run sync -- rheem   # explicit project
```

### 4. Validate

```bash
npm run validate        # prints per-segment status: OK / MISSING / PLACEHOLDER
```

### 5. Render

```bash
npm run render          # full H.264  → out/rheem-demo.mp4
npm run render:hq       # CRF 18
npm run render:preview  # half-res
npm run render:gif      # animated GIF (30 fps)
```

### Or run the complete pipeline

```bash
npm run pipeline -- https://your-app.com --user admin --pass secret
```

## Adding a new project (3 files)

**1. `projects/myapp/config/project.config.ts`**
```typescript
import type {ProjectConfig} from '../../core/types';
export const myAppProject: ProjectConfig = {
  id: 'myapp', fps: 60, width: 1920, height: 1080,
  product:  { name:'My App', tagline:'Your tagline.', ctaText:'Try My App', ctaSubtext:'myapp.com' },
  theme:    { primaryColor:'#7C3AED', accentColor:'#F59E0B' },
  recording:{ appUrl: process.env.MYAPP_URL||'', outputDir:'projects/myapp/recordings' },
  scenes:   [
    {type:'intro', durationInFrames:900}, {type:'problem', durationInFrames:1200},
    {type:'solution', durationInFrames:1200}, {type:'productDemo', durationInFrames:2100},
    {type:'features', durationInFrames:1200}, {type:'metrics', durationInFrames:1200},
    {type:'customerJourney', durationInFrames:900}, {type:'outro', durationInFrames:900},
  ],
  content: { problems:[…], features:[…], metrics:[…], customerJourney:[…] },
};
```

**2. `projects/myapp/config/segmentDefs.ts`**  
Copy from `projects/rheem/config/segmentDefs.ts`, adjust segment IDs + keywords.

**3. `projects/myapp/composition.tsx`**  
Copy from `projects/rheem/composition.tsx`, replace all `rheem` references.

**4. Register in `src/Root.tsx`**
```typescript
import {MyAppDemo} from '../projects/myapp/composition';
// add <Composition id="MyAppDemo" component={MyAppDemo} … />
```

**5. Run**
```bash
npm run record -- https://myapp.com --project myapp --user u --pass p
npm run sync -- myapp
npm run validate -- myapp
npx remotion render MyAppDemo out/myapp-demo.mp4
```

## Scene reference

| Scene | File | Props |
|---|---|---|
| SharedIntro | `core/scenes/SharedIntro.tsx` | `product: ProductInfo` |
| ProblemScene | `core/scenes/ProblemScene.tsx` | `problems: ProblemCard[]` |
| SolutionScene | `core/scenes/SolutionScene.tsx` | `nodes, edges` |
| ProductDemoScene | `core/scenes/ProductDemoScene.tsx` | `segments: DemoSegment[]` |
| FeaturesScene | `core/scenes/FeaturesScene.tsx` | `features: FeatureCard[]` |
| MetricsScene | `core/scenes/MetricsScene.tsx` | `metrics: MetricCard[]` |
| CustomerJourneyScene | `core/scenes/CustomerJourneyScene.tsx` | `steps: JourneyStep[]` |
| SharedOutro | `core/scenes/SharedOutro.tsx` | `product: ProductInfo` |

All scenes read the active theme via `useTheme()`.  
`BrandingEngine` in the composition injects the project's resolved theme automatically.

## Theme override

Set `primaryColor` and `accentColor` in `project.config.ts → theme:`.  
`resolveTheme()` in `core/themes/engine.ts` propagates them to glow/subtle/border variants automatically.

## Customizing Content

All copy, metrics, features, and scene timings live in **`src/config/videoConfig.json`**.
Edit that file — no component code changes needed for content updates.

### Adding a Screen Recording

Place your video/image in `src/assets/` and pass the path to `ScreenShowcase`:

```tsx
<ScreenShowcase
  src={staticFile('assets/dashboard-recording.mp4')}
  zoomRegions={[
    {startFrame: 120, endFrame: 300, x: 200, y: 150, width: 600, height: 300}
  ]}
  clickHighlights={[
    {frame: 150, x: 350, y: 280}
  ]}
/>
```

### Adding Captions

```tsx
import {CaptionComponent} from '../components/CaptionComponent';

<CaptionComponent captions={[
  {startFrame: 0,  endFrame: 120, text: "Welcome to Rheem", speaker: "Narrator"},
  {startFrame: 130, endFrame: 260, text: "Built for enterprise scale"},
]} />
```
