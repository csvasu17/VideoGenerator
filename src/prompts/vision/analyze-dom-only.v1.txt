## Analysis Framework

From the HTML structure and text, infer:

1. **Page Purpose** — What is this page for? Base your answer on headings, navigation labels, form fields, and visible text.

2. **Key Features** — Identify 2–5 capabilities evident from the DOM. Look for: form elements, table structures, navigation items, data labels, button text, heading hierarchy.

3. **Important Actions** — Buttons, links, and form submits that indicate key user workflows.

4. **Business Context** — Based on the terminology and structure, what business process does this page serve?

5. **KPI Widgets** — Any numeric values, percentages, or status labels in the DOM.
